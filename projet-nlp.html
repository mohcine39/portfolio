Analyse de Sentiments (NLP)

Fine-tuning d'un modèle BERT pour la classification d'avis e-commerce.

PyTorch HuggingFace Streamlit Docker

Objectif du Projet

Analyser et classifier automatiquement de grands volumes de retours clients (commentaires, avis) afin de déterminer si le sentiment est Positif, Neutre ou Négatif. Cela permet aux équipes produit d'agir rapidement en cas de pic d'insatisfaction.

Architecture & Modèle

Pour cette tâche de Traitement du Langage Naturel (NLP), j'ai choisi l'architecture Transformers en utilisant le modèle pré-entraîné CamemBERT (optimisé pour le français), fourni par la bibliothèque HuggingFace. J'ai effectué un fine-tuning sur un dataset de plus de 50 000 avis annotés.

from transformers import AutoTokenizer, AutoModelForSequenceClassification

# Chargement du modèle et tokenizer
model_name = "camembert-base"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSequenceClassification.from_pretrained(model_name, num_labels=3)

# Tokenization du texte brut
inputs = tokenizer("La livraison a été incroyablement rapide, je recommande !", 
                   return_tensors="pt")

# Inférence
outputs = model(**inputs)

Interface Utilisateur (Streamlit)

Afin de rendre ce modèle accessible aux membres non techniques de l'équipe (marketing, produit), j'ai développé une application web interactive avec Streamlit. L'interface permet d'importer un fichier CSV d'avis et d'obtenir un tableau de bord récapitulatif généré en quelques secondes.

Déploiement

L'application a été conteneurisée avec Docker pour garantir sa portabilité, puis déployée sur une instance Cloud (AWS EC2). Un système d'intégration continue (CI/CD) a été mis en place via GitHub Actions.
